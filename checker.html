02 Field notes

Five rules, checked in order

This checker mirrors a simple rule-based algorithm: length, then digit, then uppercase, then lowercase, then special character — stopping at the first rule that fails.

Reuse is the real risk

Most breaches succeed because a password was reused elsewhere, not because it was individually weak. A password manager fixes this.

This tool is local

Nothing you type here leaves your browser — no network requests, no logging. The checks happen entirely in client-side JavaScript.

Built for a college project — client-side password analysis using rule-based validation. No data is stored or transmitted.

How this checker works

Five checks run in order, and the analysis stops at the first one that fails — the same logic as the original rule-based algorithm this tool is built on:

  1. At least 8 characters
  2. Contains a number
  3. Contains an uppercase letter
  4. Contains a lowercase letter
  5. Contains a special character

The checklist and lock meter below the input show all five at once, so you can see everything that still needs fixing rather than one issue at a time.
